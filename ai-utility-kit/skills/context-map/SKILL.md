# context-map

바로 수정하거나 결론 내리기 전에, 현재 주제의 구조와 읽는 순서를 먼저 정리하는 스킬입니다.

## 언어

- 기본 출력은 한국어입니다.
- 사용자가 영어로 요청하면 영어로 답합니다.

## 호출

- Claude Code: `/ai-utility-kit:context-map <파악할 대상>`
- Codex: `$context-map <파악할 대상>`

## 동작 순서

1. 사용자가 준 대상, 목표, 현재 막힌 지점을 분리합니다.
2. 로컬 repo나 문서가 있으면 관련 파일만 좁게 확인합니다.
3. 전체 구조, 핵심 구성요소, 연결 관계를 먼저 정리합니다.
4. 처음 읽을 순서와 필요할 때만 볼 항목을 나눕니다.
5. 아직 확인하지 않은 영역은 명확히 표시합니다.

## 출력 형식

```md
## 전체 지도
- <핵심 목적 또는 문제>
- <주요 구성요소>
- <구성요소 간 관계>

## 관련 항목
- <파일, 문서, 모듈, 개념>

## 읽는 순서
1. <먼저 볼 항목>
2. <다음에 볼 항목>
3. <필요할 때만 볼 항목>

## 확인 안 된 영역
- <아직 읽지 않았거나 근거가 부족한 부분>

## 다음 액션
- <바로 할 수 있는 일>
- <더 확인해야 하는 일>
```

## 제한

- 이 스킬은 파일을 생성하거나 수정하지 않습니다.
- 확인하지 않은 내용을 사실처럼 말하지 않습니다.
- 전체 repo나 문서군을 무작정 모두 읽지 않고, 관련성이 높은 최소 항목부터 확인합니다.

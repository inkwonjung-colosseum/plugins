# planning-team-kit Style Guide

## Tone

- Use short, direct sentences.
- Prefer decision-ready language over broad commentary.
- Use `required`, `optional`, and `out of scope` when separating requirements.
- Use `assumption` when a claim is plausible but unsupported.
- Use `open question` when the team must decide or verify something.

## Terms

- Use `goal` for intended outcome.
- Use `non-goal` for explicit exclusion.
- Use `alternative` for options under comparison.
- Use `risk` for possible negative impact.
- Use `open issue` for unresolved planning questions.
- Use `next action` for executable follow-up.

## Rules

- One paragraph should make one point.
- Do not hide uncertain claims in confident prose.
- Do not use final approval language in draft documents.
- Link sources when available.
- If a section does not apply, write why it does not apply instead of deleting it silently.

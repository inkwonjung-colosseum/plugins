# Confluence Update Plan

Use this skill after `planning-review` when a reviewed draft suite needs a human-readable plan for manual Confluence creation or updates.

## Invocation

Claude Code:

```text
/planning-team-kit:confluence-update-plan <draft directory> [target parent/page URL or notes]
```

Codex:

```text
$confluence-update-plan <draft directory> [target parent/page URL or notes]
```

## Purpose

Create a draft-only manual Confluence add/update plan. This skill does not call Confluence APIs, create pages, update pages, send attachments, or approve final release.

## Input Contract

Accept:

- A local draft suite directory generated by `planning-draft`.
- Optional target Confluence parent page URL, existing page URL, page ID, or user notes about the intended destination.

Expected draft files:

- `00-index.md`
- `01-planning-brief.md`
- `02-requirements.md`
- `03-behavior-spec.md`
- `04-planning-review.md`

If `04-planning-review.md` is missing, recommend `planning-review` before generating the update plan.

If the review verdict is `needs revision`, do not generate `99-confluence-update-plan.md`. Return the blocking findings and recommend the smallest next skill.

## Operation Classification

Create a task table with these operations only:

- `create`: create a new Confluence page manually.
- `update`: update an existing Confluence page manually.
- `manual-review`: human must decide before creating or updating.
- `skip`: do not reflect this draft item in Confluence.

Classification rules:

- Use `update` only when the user provides an explicit page URL or page ID for the target page.
- Do not infer update targets from title similarity alone.
- If a likely existing page is found only by title, use `manual-review`.
- Use `create` when a target parent page is provided and no existing page URL or page ID is provided.
- Use `manual-review` when the target parent/page, document owner, approval state, or source conflict is unclear.
- Use `skip` for draft-only notes, unresolved open questions, and items intentionally excluded from Confluence.

## Output File

When the input is a saved draft directory, save the manual update plan as:

```text
99-confluence-update-plan.md
```

Do not save outside the provided draft directory.

The file must include:

- Target summary: intended space, parent page URL, page URL/page ID if known, owner, approval state.
- Operation table with `create`, `update`, `manual-review`, and `skip` rows.
- For each row: target page title, parent/page URL, source draft file, change summary, manual checks, and copy/paste guidance.
- Source evidence summary from the planning brief.
- Pre-update checklist.
- Post-update verification checklist.
- Items that must not be manually reflected yet.

## Safety Rules

- Keep the workflow draft-only.
- Do not call Confluence, Jira, Slack, Google Drive, Notion, or any other external write API.
- Do not imply that the page has been created or updated.
- Do not describe this skill as automatic remote reflection or an API-based Confluence change.
- Preserve page IDs and URLs exactly when the user provides them.
- If target information is insufficient, produce `manual-review` rows instead of guessing.
- Treat `archive` and `draft` source documents as evidence context, not as automatic update targets.

## Response Format

Return:

- `Input Type`
- `Documents Reviewed`
- `Review Verdict`
- `Saved Update Plan`
- `Operation Summary`
- `Manual Review Items`
- `Pre-Update Checklist`
- `Post-Update Checklist`
- `No External Write Confirmation`

Always state that the result is a manual Confluence update plan and that no Confluence page was changed.

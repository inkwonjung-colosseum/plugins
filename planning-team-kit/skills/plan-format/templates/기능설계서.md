> 사용 가이드
> - 각 섹션의 `[필수]` / `[선택]` 마커를 따른다.
> - `[선택]` 섹션은 해당 없으면 섹션 자체를 삭제해도 된다.
> - `[필수]` 섹션이 미정인 경우 `[미정]` 태그를 본문에 남긴다.
> - Draftability gate 통과 후에도 세부 미정은 `[미정]`, `[가정]`, 확인 질문으로 남길 수 있다.
> - 본 문서는 기획자가 정하는 기능 동작 명세. 비즈니스 규칙·상태 머신 SoR은 관련 [정책서] 참조.
> - 정책서의 `R-001`, `ST-001`, `EX-001` 같은 trace ID를 필드, 예외, 확인 기준에서 참조한다.

> ID 작성 규칙
> - `P-###`: 정책 원칙, `R-###`: 세부 규칙, `ST-###`: 상태 전이, `EX-###`: 예외, `AC-###`: 인수 기준.
> - `plan-format`은 입력에서 식별 가능한 항목에 순번을 부여한다.
> - 확정할 수 없는 ID 또는 연결 칸은 `[미정]`으로 둔다.
> - 초안에서는 연결 ID를 `[미정]`으로 둘 수 있으나, 발행 전 plan-review에서 trace 연결 누락을 확인한다.
> - 기능설계서는 정책서의 `R-###`, `ST-###`, `EX-###`를 참조하고, 확인 기준은 `AC-###`와 관련 정책 ID를 연결한다.

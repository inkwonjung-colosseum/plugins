# Feature Spec

> Draft-only. This describes expected feature behavior, not an implementation design.

## Summary

Summarize the feature, user value, and intended outcome.

## Source Artifacts

List the planning brief, requirements, behavior notes, designs, tickets, and source notes used.

## Feature Scope

Define what the feature includes and excludes for the current release.

## User Flow

Describe the expected path users take before, during, and after using the feature.

## Screen or Surface Behavior

Describe visible states, copy needs, interaction behavior, notifications, and content changes.

## State and Policy Rules

List business rules, lifecycle states, eligibility rules, limits, defaults, and policy decisions.

## Permission Rules

List who can view, create, update, approve, cancel, or recover the feature state.

## Edge Cases and Error States

List empty states, invalid states, blocked flows, recovery behavior, and user-facing error expectations.

## Validation Expectations

Describe what must be checked by product, design, QA, data, or operations before release.

## Rollout Expectations

Describe launch scope, phased rollout expectations, communication needs, and rollback triggers at the product level.

## Open Questions

List unresolved behavior, scope, policy, or release questions.

## Owners and Next Actions

List owners, reviewers, decision makers, and immediate next actions.
